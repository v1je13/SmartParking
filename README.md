# АСУ Умная парковка города

## Описание
Система управления автоматизированной парковкой на C# с использованием:
- **Avalonia UI 11** - кроссплатформенный интерфейс
- **Entity Framework Core** - работа с БД
- **SQL Server (LocalDB)** - база данных
- **Microsoft Office Interop.Word** - печать квитанций

## Установка и запуск

### Требования
- **Visual Studio 2022** или **VS Code** с C# расширением
- **.NET 8 SDK**
- **SQL Server LocalDB** (устанавливается с Visual Studio)

### Шаги

1. **Откройте проект в Visual Studio:**
   ```
   File → Open → Folder (выберите папку SmartParking)
   ```

2. **Восстановите NuGet пакеты:**
   ```bash
   dotnet restore
   ```

3. **Создайте базу данных:**
   - В Package Manager Console выполните:
   ```bash
   Update-Database
   ```
   
   Или просто запустите приложение - БД создастся автоматически.

4. **Запустите приложение:**
   ```bash
   dotnet run
   ```
   
   Или нажмите **F5** в Visual Studio.

## Функции

### 📊 Главная панель (Dashboard)
- Статистика по паркингам
- Количество свободных/занятых мест
- Активные сессии парковки
- Информация о водителях и должниках

### 📍 Парковочные зоны
- Добавление/редактирование зон
- Просмотр и удаление
- Настройка цены и режима работы

### 🅿 Парковочные места
- Управление местами в каждой зоне
- Изменение статуса (свободно/занято/ремонт)
- Типы мест (обычное, для инвалидов, для грузовых)

### 👤 Водители и машины
- Регистрация водителей
- Добавление/удаление машин
- Отслеживание владельцев

### 🚗 Заезды/Выезды
- Ручная регистрация въезда машины
- Отметить выезд
- Отмечать оплату
- Выписывать штрафы
- Отправлять на эвакуацию

### 📊 Отчёты
- **✅ Свободные места в зоне** - сводка по всем зонам
- **🚗 Занятые места** - какая машина на каком месте
- **📋 История машины** - все парковки конкретного авто
- **⚠️ Должники** - неоплаченные парковки
- **📊 Загрузка зон** - зоны с загруженностью >90%
- **⏱ Среднее время** - статистика по времени стоянки
- **💰 Выручка** - выручка по зонам
- **🚔 Эвакуированные** - машины отправленные на эвакуацию

### 🖨 Печать квитанций
- Просмотр квитанции в Word
- Печать в Word
- Заполнение реквизитов автоматически

## Тестовые данные

При первом запуске приложение автоматически добавит:
- **3 парковочные зоны** (Центральная, Торговый центр, Подземная у вокзала)
- **30+ парковочных мест**
- **3 водителей** с машинами
- **Несколько тестовых сессий парковки**

## Структура БД

### Таблица ParkingZones
- Id (PK)
- Name - название зоны
- Address - адрес
- ZoneType - тип (платная, бесплатная, подземная)
- TotalSpots - всего мест
- PricePerHour - цена за час
- WorkingHours - режим работы
- IsActive - активна

### Таблица ParkingSpots
- Id (PK)
- ParkingZoneId (FK)
- SpotNumber - номер места (А1, Б2 и т.д.)
- SpotType - тип места
- Status - статус (Free, Occupied, Repair)
- Notes - примечания

### Таблица Drivers
- Id (PK)
- LastName, FirstName, MiddleName
- Phone - телефон
- Email
- DriverLicense - номер водительского
- RegisteredAt - дата регистрации
- HasDebt - есть ли долг

### Таблица Vehicles
- Id (PK)
- DriverId (FK)
- LicensePlate - гос. номер
- Brand, Model - марка и модель
- Color - цвет
- Year - год выпуска
- VehicleType - тип машины

### Таблица ParkingSessions
- Id (PK)
- VehicleId (FK) - машина
- ParkingSpotId (FK) - место
- EntryTime - время заезда
- ExitTime - время выезда
- AmountDue - сумма к оплате
- AmountPaid - сумма оплачена
- IsPaid - оплачено ли
- FineAmount - размер штрафа
- FineReason - причина штрафа
- Status - статус сессии
- Notes - примечания оператора
- IsSubscription - абонемент
- SubscriptionExpiry - до когда абонемент

## Строка подключения

По умолчанию используется LocalDB:
```
Server=(localdb)\mssqllocaldb;Database=SmartParkingDB;Trusted_Connection=True;
```

Если хотите использовать другой сервер, отредактируйте в `Data/AppDbContext.cs`:
```csharp
optionsBuilder.UseSqlServer("ваша_строка_подключения");
```

## Печать квитанций

Для печати нужен Microsoft Word (2010+).

При нажатии на кнопку "Печать" квитанция:
1. Открывается в Word
2. Заполняется автоматически
3. Отправляется на печать

Если Word не установлен, получите ошибку в консоли.

## Возможные ошибки и решения

### "Could not connect to SQL Server"
- Убедитесь, что установлена SQL Server LocalDB
- В Visual Studio: Tools → SQL Server → Локальная база (LocalDB)

### "Cannot write to database"
- Удалите папку `bin` и `obj`
- Выполните `dotnet clean` и `dotnet build`

### "Word не открывается при печати"
- Установите Microsoft Office
- Или используйте функцию просмотра без Word
